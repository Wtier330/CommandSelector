# 运维工具界面：自适应布局与动态缩放（页面设计规范，桌面优先）

## 1) Layout（布局系统与逻辑）

* 总体采用 **“外壳 + 可缩放画布”** 混合方案：

  * 外壳（App Shell）：负责最小窗口保护态提示、抽屉/弹层层级、全局快捷键等。

  * 画布（Canvas）：承载主界面内容，使用 CSS Grid/Flex 做响应式，并通过 `scaleRatio(0.8–1.2)` 做密度微调。

* 主结构建议：

  * 顶部栏（Topbar）：固定高度，包含关键操作（P0）。

  * 主区（Main）：左导航（Sider）+ 内容区（Content）+（可选）辅助区（Aside）。

* 响应式策略：

  * **先断点变形（列/面板/导航形态）**，再应用 scaleRatio。

  * 内容区内部优先使用 **CSS Grid（12 列）**；在 S 断点退化为单列。

## 2) Meta Information（页面级元信息）

* Title：运维工具

* Description：提供运维操作入口、状态与信息展示，支持自适应布局与动态缩放。

* Open Graph：

  * og:title = 运维工具

  * og:description = 自适应布局与动态缩放

## 3) Global Styles（全局样式与设计 Tokens）

* 设计基调：信息密集但可读，强调可扫描性。

* 颜色（示例 token 名称，可映射到现有主题系统）：

  * `--bg`：#0B1220（深色背景）

  * `--panel`：#111A2E（面板底色）

  * `--text`：#E6EAF2（主文字）

  * `--muted`：#98A2B3（次级文字）

  * `--accent`：#3B82F6（强调色/主按钮）

  * `--danger`：#EF4444（危险操作）

* 字体与排版：

  * 基础字号 `--font-base` 建议 14px；紧凑态可降至 13px（但不建议低于 12px）。

  * 标题层级：H1/H2/H3 递减，搭配字重区分。

* 交互样式：

  * Button：默认/hover/active/disabled 明确区分；危险操作使用 `--danger`。

  * Link：下划线仅在 hover 出现，减少视觉噪声。

* 动效：

  * 抽屉/弹层进入：150–200ms ease-out；避免夸张动画影响效率。

## 4) Page Structure（页面整体结构）

### 页面：运维工具主界面

* 布局骨架：

  1. Topbar（固定）
  2. Main（三栏可变形：Sider + Content + Aside）
  3. Status/Toast（右下角或顶部）

## 5) Sections & Components（分区与组件细则）

### 5.1 Topbar（P0，不可折叠）

* 左侧：产品名/当前环境（文本过长省略号）。

* 中部：关键操作区（主按钮 1–3 个，必要时收纳到“更多”菜单）。

* 右侧：用户/设置入口（如存在）。

* 紧凑策略：

  * M/S 断点：按钮图标化 + Tooltip；保留最多 1 个主按钮为“文字+图标”。

### 5.2 Sider（左导航，P3，可折叠）

* XL/L：宽导航（带文字）。

* M：图标导航（仅图标 + Tooltip）。

* S：默认隐藏为抽屉（左上角汉堡按钮打开）。

### 5.3 Content（主内容，P0/P1 混合，优先保留）

* 内容区采用 12 列 Grid：

  * XL：可 3 列卡片/2 列主从布局（由业务内容决定）

  * L：2 列为主

  * M：1–2 列自适应

  * S：单列堆叠

* 组件建议（不绑定具体业务，仅规定容器能力）：

  * 信息卡片（Card）：标题/状态角标/操作区（右上）。

  * 表格/日志列表（Table/List）：支持固定表头、行高紧凑切换（H2/H1）。

  * 筛选条（Filter Bar）：可折叠为“筛选”按钮弹出面板。

### 5.4 Aside（右侧辅助面板，P1，优先折叠）

* XL：常驻右侧（例如详情、帮助、说明等）。

* L/M：默认折叠为抽屉（从右侧滑出）。

* S：默认使用全屏弹层（避免挤压主内容）。

### 5.5 Status / Toast（提示区，P0）

* 成功/失败提示不遮挡主操作。

* “窗口过小”提示：

  * 当 viewport < 800×600：显示固定横幅或半透明遮罩说明；提供“我知道了”按钮，但不继续降低信息可读性。

## 6) Breakpoints（断点与折叠规则：页面侧可视化准则）

### 6.1 宽度断点

* XL：>=1440

* L：1200–1439

* M：960–1199

* S：800–959

* 不支持：<800

### 6.2 高度阈值

* H2：>=800（正常密度）

* H1：600–799（紧凑密度）

* 不支持：<600

### 6.3 折叠优先级（从先到后）

1. 右侧辅助面板 Aside（侧栏 -> 抽屉 -> 弹层）
2. 次要信息区块（卡片组/说明区）变 Accordion，默认收起
3. 左导航 Sider（宽 -> 图标 -> 抽屉）
4. 非关键文案/装饰直接隐藏
5. 最后才允许局部滚动（保持 scaleRatio 不低于 0.8）

## 7) Dynamic Scaling（动态缩放：交互与可用性规则）

* 缩放范围：scaleRatio 0.8–1.2。

* 应用顺序：断点变形 -> 折叠 -> 缩放（必要时再触发一次折叠校验）。

* 可读性底线：

  * 不允许为了“塞下全部内容”把缩放降到 0.8 以下。

  * 当理论 rawScale < 0.8：固定 0.8，并通过折叠与局部滚动解决。

* 建议提供（可选）缩放控制入口：

  * 仅作为“密度微调”，默认自动；手动调整时仍需 clamp 到 0.8–1.2。
