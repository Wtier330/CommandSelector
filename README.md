# CommandSelector

> 让运维命令管理变得优雅而高效

## 前言

凌晨两点四十七分。

手机震动。我闭着眼睛摸过来，瞥了一眼屏幕——**老万**。

老万是今晚值班的运维工程师。我们轮流排班，他今晚运气不好。

接通电话，那头的声音急促又压抑："出事了，P0，支付系统全挂了。用户都登不进去，CTO已经在群里炸了，客户那边在问赔偿的事。你那个集群扩容的脚本还有吗？"

我挂掉电话，裹着被子坐在床边。脑子里第一反应不是"怎么解决"，是——

**那个脚本叫什么来着？**

电脑在客厅，媳妇带着孩子睡在卧室。我蹑手蹑脚走到客厅，光脚踩在地板上，冰的。

开机。12秒。

我开始拼命回忆。

`cluster-scale`？不对，好像是 `scale-up`？或者是 `auto_scale`？上次老万说"集群扩容"的时候，我随手存的，名字到底是什么来着？

我先翻飞书。搜索"集群"。没有。搜索"扩容"。没有。搜索"scale"。出来 700 条记录，挨个点开——妈的我怎么知道哪个是老万发的那个。

翻微信。翻钉钉。翻邮箱。

对了，那天王涛在群里发过一个脚本，说是"测试好了"——王涛是叫王滔还是王掏？群是叫"运维群"还是"技术群"？我搜了"脚本"，出来 200 条。

我记起来了，我把它放在桌面上了。

桌面上有 38 个图标。我一个个看。没有。回收站呢？没有。

我开始怀疑自己是不是把它放在网盘里了。打开网盘，同步了 3 分钟。搜索"集群"。同上的 700 条。

……

凌晨三点二十一分。老万又打过来："你找到没？CTO在问具体什么时候能恢复。"

"马上马上。"

我嘴上说着，心里知道——"马上"是个谎言。

……

找到了。在飞书一个叫"YY-项目-2024"的历史群里，文件名是 `集群扩容命令改版最终稳定增强版（勿动）(4).py`。

我小心翼翼地打开它。扫了一眼。有点眼熟，但不确定这是不是我上次改的那版。脚本里有行注释写着 `REM by witer 2023`，但我明明记得我没改过这行……

**管它呢。先跑。**

复制参数，执行。

```
Error: Path format not supported
```

我把脚本里的路径格式从 `/` 改成 `\`，再跑。

```
Error: Access denied
```

我盯着屏幕。额头有点痒。

老万又打过来："情况恶化了，订单数据开始出现延迟，CEO已经在打电话了。"

我深吸一口气，把错误信息发给老万。

"脚本有bug，你先用手动扩容顶一下，我这边……我再找找。"

电话那头沉默了两秒。老万没骂我，只是说："快点。"

挂了电话。

---

**三年了。我的脚本散落在 6 个聊天软件、2 个云笔记、1 个共享盘、还有不知道谁的电脑里。**

每次故障来的时候，我不是在想怎么解决——**我先要想那个该死的脚本叫什么、在哪、能不能用。**

而老万挂掉电话的那一刻，我知道他也一样无奈——他没有选择，只能等一个他根本不确定什么时候能到的脚本。

---

直到那天。

我对键盘说："我tm受够了。"

于是，CommandSelector 诞生了。

**它不是要替代终端。它只是想回答：当我想用那个命令的时候，它应该在那里——3秒之内。**

## 什么是 CommandSelector？

CommandSelector 是一款专为运维工程师设计的**命令和脚本管理工具**。

它解决的不是"如何执行命令"的问题——终端已经做得很好了。它解决的是**如何组织、管理、和快速找到**那些你每天都在用的命令和脚本。

### 你是否遇到过这些困扰？

- **找不到**：上次用的那个命令存在哪里了？共享盘？云笔记？还是小王发的那封邮件？
- **记不住**：脚本的参数顺序是什么来着？用 `@echo` 还是 `Write-Host`？
- **不敢改**：这个生产环境的脚本用了三年了，万一改坏了怎么办？
- **难复用**：本地调试好的脚本，怎么快速分享给团队？

这些问题，CommandSelector 都有针对性的解决方案。

## 核心特性

### 命令库管理

将所有命令集中存储在一个地方。支持分类、标签、搜索，随时随地访问。

### 参数化模板

告别硬编码。使用 `{{参数名}}` 占位符定义模板，每次使用时填写参数值，生成最终命令。减少复制粘贴的错误。

### 脚本元数据解析

导入本地脚本文件（BAT/PS1/VBS/SH/CMD/PY），自动解析脚本头部注释中的元数据：名称、分类、参数说明、使用示例。

### 拼音搜索

输入 `cwgl` 就能找到"磁盘管理"。支持精确匹配、拼音首字母匹配、模糊匹配。

### AI 智能补全

配置你的 AI 服务，让 AI 自动为命令补全描述、分类、标签。批量导入脚本时，再也不用一个个手动填写元数据了。

### 双引擎支持

同一套模板，支持 CMD 和 PowerShell 两种格式。根据目标环境一键切换。

### 回收站

误删的命令不会消失。进入回收站，随时恢复或彻底删除。30 天自动清理，保持库的健康。

## 快速开始

### 下载安装

从 [Releases](https://github.com/witer330/CommandSelector/releases) 页面下载最新版本的安装包，运行安装即可。

### 添加第一个命令

1. 点击左侧工具栏的 **「+」** 按钮
2. 填写命令名称和描述
3. 在命令模板中输入你的命令，使用 `{{参数名}}` 定义占位符
4. 保存

### 导入脚本

1. 点击工具栏的 **「导入脚本」** 按钮
2. 选择本地脚本文件
3. CommandSelector 自动解析脚本头部的元数据注释
4. 确认导入

### 使用命令

1. 在搜索框中输入关键词（支持拼音）
2. 找到目标命令后，点击使用
3. 填写参数值，点击复制或直接执行
