## 1. 背景与使用场景

- 我在实际运维中**只能通过远程 CMD** 对网吧/客户机进行维护处理。
- 要求 **静默**，不能影响正在使用机器的用户（不弹窗、不打断）。
- 有两类常见命令形态：
  1. **纯 CMD/批处理**（例如 netstat、sc、ipconfig 等）
  2. **CMD 中调用 PowerShell**（例如通过 COM 读取桌面快捷方式 `.lnk` 的 TargetPath/Arguments，判断是否被劫持）
- 我希望把这些命令形成一个**企业内部开放的知识库/命令库**，供各部门都能访问、搜索、复制，并能在复制前**修改命令中的关键参数**。

------

## 2. 总体目标

做一套单体本地工具（Tauri + Vue3）：

- 本地维护命令知识库（新增/编辑/删除）。
- 提供搜索、参数编辑、命令预览与一键复制。
- 不提供执行功能（只复制，不执行）。

------

## 3. 核心功能需求

### 3.1 浏览与复制（必须）

- 命令库列表：
  - 搜索（按名称/描述/标签）
  - 查看命令详情
- 参数修改入口：
  - 在详情页展示该命令的参数表单（如路径、端口、PID 等）
  - 用户修改参数后，实时渲染最终命令
- 复制能力：
  - 一键复制“最终渲染后的命令”（用于远程 CMD 下发）
- 校验与防呆（建议 V1 必做）：
  - required 参数不能为空
  - path 参数可做后缀校验（如 `.lnk`）
  - 渲染后若仍残留 `{{KEY}}` 占位符，禁止复制并提示原因
- 本地记忆：
  - 记住用户对某条命令最近一次填写的参数值（下次打开自动回填）

> 说明：仅生成可复制命令，不执行、不检查运行结果。

------

### 3.2 命令维护（必须）

- 本地存储编辑态数据（可先用 `library.json`，后续可演进 SQLite）。
- 命令录入/编辑：
  - 可粘贴完整命令文本（含 CMD 调 PowerShell 的一行命令）
  - 支持把命令中的某段文本设为“可修改参数”
- **方案 B（重点）：选中文本一键提取为参数**
  - 在编辑器中选中一段内容（例如 `C:\\Users\\Administrator\\Desktop\\谷歌浏览器.lnk`）
  - 点“提取为参数”
  - 弹窗填写/确认参数 key、类型、默认值、校验
  - 自动将选中文本替换为 `{{KEY}}`
  - 自动写入/更新参数定义
  - 处理 key 冲突（复用/新建）
## 4. 数据存储方式

- 统一使用本地 `library.json` 作为当前数据源。
- 后续需要更强的数据维护能力时，再演进到 SQLite。

------

## 5. 命令模板与参数化的表现形式

- 命令模板使用占位符：`{{KEY}}`（例如 `{{LNK_PATH}}`）
- 参数定义包含：
  - key、label、type、default_value、required、validation、hint
- 客户端渲染逻辑：
  - 用用户填写值替换占位符，生成最终命令行（用于远程 CMD 下发）
- 典型示例（检查快捷方式是否被劫持）：
  - 用途：检查桌标快捷方式内容
  - 可修改参数点：`.lnk` 路径
  - 最终复制内容为一行 CMD 可执行命令（CMD 调 powershell -Command ...）

------

## 6. 明确不做的内容（当前阶段）

- 不做：命令执行（本地/远程）
- 不做：权限、分组、审批流
- 不做：复杂的脚本运行检测（“检查是否能运行”先不纳入这一阶段）
- 不做：增量更新、签名校验（可作为后续增强）

------

## 7. 阶段性计划（已决定的实现顺序）

- 第一步：完善本地编辑 + 导入/导出 + 参数编辑复制闭环
- 第二步：再做“快速编写/复杂拼接”（例如提权、静默、EncodedCommand 等增强）
